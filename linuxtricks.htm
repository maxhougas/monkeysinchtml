Regex

"sed 's:[REGEX]:[TOKEN]:g'": global replace; use -i to edit files in place.

"sed 's:[REGEX]:[TOKEN0]&[TOKEN1]:g'": replace all [REGEX] with [TOKEN0][REGEX][TOKEN1].

"sed '$s:[REGEX]:[TOKEN]:g'": replace [REGEX] in last line.

"sed 's:[REGEX]:[TOKEN]:g; s:[REGEX1]:[TOKEN1]:g'" multilple sed commands can be run from a single invocation.

"grep [REGEX]" grab line with [REGEX]; use -i to ignore case; use -o to grab tokens instead of lines.

Look ahead and look behind only work with grep -P.

"[a-zA-Z0-9._-]*" is a handy regex.

"https?://[A-Za-z0-9/%=\'\"\\._-]*" might grab URLs.

Link

"ln -sT [TARGET] [LINK]": create [LINK] to [TARGET].

Automation

"yes [TOKEN] | [PROG]": will automatically answer [TOKEN] when [PROG] asks for user input. Handy when -y is not an option.

"[PROG] < [FILE]": is handy when you need different answers for different inputs.

"CMD [PROG] && tail -f /dev/null": is good for containers when [PROG] must be backgrounded.

A script containing "#!/bin/sh tail -f /dev/null" will ensure that tail is running in an sh subshell.
